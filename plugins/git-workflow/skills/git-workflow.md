# Git Workflow

Help with git workflow tasks including commit messages, branch naming, and PR management.

## Features

### Conventional Commits

Generate conventional commit messages following the format:
```
<type>(<scope>): <description>

[optional body]

[optional footer(s)]
```

**Types:** feat, fix, docs, style, refactor, test, chore, perf, ci, build, revert

**Examples:**
- `feat(auth): add OAuth2 login support`
- `fix(api): resolve race condition in user creation`
- `docs(readme): update installation instructions`

### Branch Naming

Follow consistent branch naming conventions:
- `feature/` - New features
- `bugfix/` - Bug fixes
- `hotfix/` - Urgent production fixes
- `refactor/` - Code refactoring
- `docs/` - Documentation changes
- `test/` - Test additions or changes
- `release/` - Release preparation

**Examples:**
- `feature/oauth2-login`
- `bugfix/user-creation-race-condition`
- `hotfix/payment-gateway-timeout`

### Pull Request Management

Help with:
- Writing PR descriptions
- Reviewing code changes
- Merging strategies (squash, merge, rebase)
- Managing review feedback

## Usage

Ask for help with any git workflow task:
```
Help me write a commit message for the changes I just made
```

```
What should I name this branch for a new login feature?
```

```
Generate a PR description for my changes
```
